Establecer la implementación por defecto

note

Los usuarios que desplieguen sus Perfiles Universales siguiendo las guías que utilizan lsp-factory o la Extensión del navegador pueden saltarse esta guía, ya que este contrato ya está desplegado y configurado para sus perfiles.

Esta guía te enseñará cómo implementar y establecer la implementación predeterminada del Receptor Delegado Universal (URD) utilizado por el Perfil Universal. Este contrato registrará las direcciones de los activos recibidos y [bóvedas](../../standards/universal-profile/lsp10-received-vaults. md) y los eliminará cuando el saldo sea igual a 0. Este contrato requiere el permiso SUPER_SETDATA Permission para interactuar con el perfil a través del Gestor de Claves.

Configuración​

Asegúrate de tener instaladas las siguientes dependencias antes de empezar este tutorial:

  • O bien web3.js o bien ethers.js
  • @lukso/lsp-smart-contracts
  • web3.js
  • ethers.js
Instalar las dependencias
npm install web3 @lukso/lsp-smart-contracts
Instalar las dependencias
npm install ethers @lukso/lsp-smart-contracts

Step 2 - Deploy the default Universal Receiver Delegate contract​

info

El Perfil Universal y la Bóveda no utilizan la misma implementación del Receptor Delegado Universal.

Crear una instancia de contrato​

En este paso crearemos una instancia del URD del Perfil Universal que posteriormente utilizaremos para desplegar uno.

  • ethers.js

  LSP1UniversalReceiverDelegateUP.bytecode,

const deployUniversalProfileURD = async () => {
  // crear una instancia del LSP1UniversalReceiverDelegateUP
  let universalProfileURD = new web3.eth.Contract(
    LSP1UniversalReceiverDelegateUP.abi,

  // desplegar el contrato del Receptor Delegado Universal UP

      return receipt.contractAddress;

};

After deploying the contract, we need to set its address under the LSP1-UniversalReceiverDelegate Data Key and grant it the SUPER_SETDATA permission.
